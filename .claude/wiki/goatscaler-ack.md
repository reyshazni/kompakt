# GOATScaler (ACK)

Alibaba ACK's managed autoscaler. Runs in the control plane (no user-visible pod in kube-system).

## Key difference from upstream CA

GOATScaler does NOT write `cluster-autoscaler-status` ConfigMap. Kompakt's `ClusterAutoscalerDetector` returns nothing on ACK.

## What GOATScaler writes

### Events (on pods)

**ProvisionNode** -- emitted when GOATScaler decides to scale up.

```
reason: ProvisionNode
source.component: GOATScaler
message: "Provision node asa-k1abq7t6148jm1hukffb in Zone: ap-southeast-5a with InstanceType: ecs.gn8is.4xlarge, Triggered time 2026-05-20 13:31:07.774"
```

Fields parseable from message:
- Node name: `asa-k1abq7t6148jm1hukffb`
- Zone: `ap-southeast-5a`
- Instance type: `ecs.gn8is.4xlarge`
- Triggered time: `2026-05-20 13:31:07.774`

This is the earliest signal that a node is coming. Fires before ECS API call, before Node object exists.

**NotTriggerScaleUp** -- emitted when GOATScaler cannot scale up.

```
reason: NotTriggerScaleUp
source.component: GOATScaler
message: "pod didn't trigger scale-up due to missing matching nodepool: 7 NodePool NoStock, 12 NodePool ResourceNotMatch, 38 TaintToleration, 1 NodeAffinity"
```

Reasons seen in the wild:
- `NodePool NoStock` -- ECS instance type out of stock in the zone
- `NodePool ResourceNotMatch` -- nodepool instance type cannot fit the pod's requests
- `TaintToleration` -- pod does not tolerate nodepool's taints
- `NodeAffinity` -- pod's node affinity/selector does not match nodepool
- `pod has unbound immediate PersistentVolumeClaims` -- PVC not bound yet
- `VolumeBinding` -- volume zone constraints

### ConfigMaps

- `autoscaler-meta` -- scaler configuration (scan interval, thresholds). No scale-up state.
- `np<hash>-inventory-status` -- one per nodepool (~133 in a real cluster). Contains ECS instance stock availability per zone. Not scale-up events.

## Node naming

ACK node names are IP-based: `cn-jakarta.172.16.x.x`. Not useful for template matching by name prefix.

Nodepool IDs are opaque hashes: `npa7a3c05295974b56ab23537d94e74d08`. Generated by ACK, not in Terraform input, different per cluster.

## Node labels available on NotReady nodes

Set by cloud controller during registration, before kubelet marks Ready:
- `alibabacloud.com/nodepool-id`: opaque nodepool hash
- `node.kubernetes.io/instance-type`: ECS instance type (e.g., `ecs.gn7i-c16g1.4xlarge`)
- `topology.kubernetes.io/zone`: availability zone
